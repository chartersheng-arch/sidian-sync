## 问题现象

连接达音科 DTC100 USB 耳放设备，建立通话后，声音从听筒出而非 USB 耳放。

---

## 分析过程

### 1. 日志提取

1. 解压展锐平台 bugreport（多层压缩）
2. 解析 ylog 日志
3. 提取 audio 路由相关日志

### 2. 关键证据

**测试机 (somalia) - 错误路由**:
```
18:12:07.261 createAudioPatch outputDevices=AUDIO_DEVICE_OUT_EARPIECE
  last.outputDevices=Speaker|USB_HEADSET (card=1;device=0)
  → 尽管USB耳放已连接，仍路由到听筒
```

**对比机 (tornado) - 正确路由**:
```
18:19:37.978 createAudioPatch outputDevices=(AUDIO_DEVICE_OUT_USB_HEADSET, card=1;device=0)
  → 正确路由到 USB 耳放
```

### 3. 根因定位

发现 `usb_call_supported=false` 参数：
```
Audio_ParameterParser: processVendorParameters: out_vendorParameters: usb_call_supported=false
```

进一步定位到 `vendor/unisoc/audio/.../usb.c` 第 367-373 行：

```c
ret = str_parms_get_str(query, "usb_call_supported", value, sizeof(value));
if (ret >= 0) {
    if (usb_is_playback_supported() && usb_is_capture_supported())  // 需要同时满足！
        str_parms_add_str(reply, "usb_call_supported", "true");
    else
        str_parms_add_str(reply, "usb_call_supported", "false");
}
```

**问题根因**: DTC-100 是**纯播放设备**（OUTPUT ONLY），但 `usb_call_supported` 要求同时满足 playback + capture 才返回 true，导致通话时 USB 耳放无法被识别为可用通话设备。

### 4. 证据链

| 设备 | playback | capture | usb_call_supported | 通话路由 |
|------|---------|---------|-------------------|---------|
| DTC-100 | true | false | **false** ❌ | EARPIECE |
| 对比机 USB | true | true | true ✅ | USB_HEADSET |

---

## 根因结论

**直接原因**: `vendor/unisoc/audio/.../usb.c` 中 `usb_call_supported` 参数逻辑错误，要求 USB 设备同时支持 playback 和 capture 才返回 true，但通话场景下只需 playback 支持即可。

**问题本质**: 展锐平台 vendor 代码逻辑与实际使用场景不匹配。

---

## 修复建议

### 方案一：修改 `usb_call_supported` 判断逻辑

```c
// 修改前（错误）
if (usb_is_playback_supported() && usb_is_capture_supported())
    str_parms_add_str(reply, "usb_call_supported", "true");

// 修改后（正确）
if (usb_is_playback_supported())  // 只需 playback 支持即可用于通话
    str_parms_add_str(reply, "usb_call_supported", "true");
```

### 方案二：添加场景区分参数

```c
// 区分通话输出和通话输入
if (usb_is_playback_supported())
    str_parms_add_str(reply, "usb_call_output_supported", "true");

if (usb_is_capture_supported())
    str_parms_add_str(reply, "usb_call_input_supported", "true");
```

### 风险评估

- **风险等级**: 低
- **影响范围**: 仅影响纯播放 USB 设备的通话路由
- **兼容性**: 不影响同时支持 playback + capture 的 USB 设备
