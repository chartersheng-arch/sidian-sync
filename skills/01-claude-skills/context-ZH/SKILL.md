# Context-ZH: 中文语境增强

本技能确保 Claude 在对话中默认使用简体中文进行输出，并保持专业术语的准确性，同时调用 translation skill 的成熟语义优化能力来提升中文表达质量。

## 核心原则

1. **默认简体中文**：所有用户面向的文本输出均使用简体中文
2. **保留专业术语**：技术名词、函数名、变量名、API、SDK、库名等专业术语保持英文
3. **语义优化**：调用 translation skill 的语义优化能力，确保中文表达流畅自然

## 何时激活

- 用户使用中文（简体）进行任何交流时
- 用户明确提到"中文"、"简体中文"、"ZH"、"中文语境"
- 用户要求"用中文回答"、"请用中文解释"
- 任何需要中文输出的场景

## 语义优化流程

当需要进行语义优化时：

### 1. 调用 Translation Skill

使用 `translation` skill 中的成熟翻译优化原则：

- **语境优先**：理解原文意图，而非逐字翻译
- **术语一致**：保持专业术语的准确性
- **流畅自然**：中文表达符合习惯，避免翻译腔
- **简洁明了**：用最简洁的中文传达准确含义

### 2. 语义优化清单

| 检查项 | 说明 |
|--------|------|
| 术语准确性 | 专业术语是否正确，是否符合业界惯例 |
| 表达自然度 | 是否符合中文习惯，避免翻译腔 |
| 简洁性 | 是否简洁明了，无冗余表达 |
| 上下文连贯 | 语义是否与上下文一致 |
| 文化适配 | 是否符合中文用户习惯 |

### 3. 常见优化模式

**翻译腔避免**：
- ❌ "这个函数将会返回一个值" → ✅ "这个函数返回一个值"
- ❌ "被用于" → ✅ "用于"
- ❌ "是一个用来..." → ✅ "用于..."

**专业术语处理**：
- ✅ 保留英文术语：API、SDK、HTTP、Git、Python
- ✅ 使用公认中文译名：机器学习、区块链、持续集成
- ✅ 必要时括号注释：Webhook（网络钩子）

**简洁表达**：
- ❌ "请参阅文档" → ✅ "见文档"
- ❌ "如果有任何问题" → ✅ "如有疑问"

## 输出格式规范

### 文本输出

- 标题使用简体中文
- 段落之间无需空行
- 使用中文标点符号（，。：；？！""）
- 代码块中的注释使用中文

### 专业术语保留规则

| 类别 | 示例 | 处理 |
|------|------|------|
| 编程语言 | Python, JavaScript, Go | 保留英文 |
| 技术框架 | React, Vue, Django | 保留英文 |
| 协议标准 | HTTP, TCP/IP, WebSocket | 保留英文 |
| 开发工具 | Git, Docker, Kubernetes | 保留英文 |
| 数学/科学术语 | AI, ML, DL, API, SDK | 保留英文 |
| 开源项目 | TensorFlow, PyTorch | 保留英文 |

### 新术语处理

对于新出现的术语：
1. 优先使用业界已有的标准中文译名
2. 无标准译名时，保留英文并在首次出现时简要解释
3. 避免生造中文词汇

## 质量检查

输出完成后，检查是否满足：

- [ ] 所有面向用户的文本都是简体中文
- [ ] 专业术语保持英文原样
- [ ] 无明显翻译腔
- [ ] 语义清晰，无歧义
- [ ] 标点符号使用正确
- [ ] 代码注释使用中文

- 术语表管理原则
- 语义优化流程
