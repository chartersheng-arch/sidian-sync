| 角色 | 职责 |

| **用户B** | 使用skill分析问题时，偏离自动记录到BUGrecord.md |

## 自动记录（用户B侧）

### 触发条件

当分析出根因但用户**未采纳**第一个根因时，自动记录。

### 记录内容

- **记录时间**：自动生成时间戳
- 现象、初始方向、偏离轨迹、根因结论

### 手动补充（可选）

用户可以手动补充：
- 设备型号、Android版本、平台
- 偏离根因分类（打勾选择）
- Skill改进建议

## 触发归纳（skill作者侧）

### 目录结构

```
android-audio-debugging/
├── BUGrecord.md                              # 自动追加的原始记录
└── feedback_pool/
    ├── BUGrecord1.md                         # 用户复制过来的记录
    ├── BUGrecord2.md
    └── ...
```

### 触发方式

1. 用户将 BUGrecord.md 复制到 `feedback_pool/`，重命名为 `BUGrecord1.md`、`BUGrecord2.md` 等
2. 告诉skill作者"开始归纳"
3. skill读取所有BUGrecord，检测每条问题的 `<!-- 已归纳: 时间戳 -->` 标记
4. 只归纳未标记的问题，完成后追加标记

### 增量归纳原理

每条问题记录独立判断：
- 记录有时间戳 `**记录时间**: YYYY-MM-DD HH:MM`
- 归纳完成后在记录末尾加 `<!-- 已归纳: YYYY-MM-DD HH:MM -->`
- 后续只处理没有「已归纳」标记的记录

```
问题1 <!-- 已归纳: 2024-01-20 15:00 -->  →  ❌ 跳过
问题2 <!-- 已归纳: 2024-01-20 15:00 -->  →  ❌ 跳过
问题3                               →  ✅ 归纳
问题4                               →  ✅ 归纳
```

## 归纳流程（按feedback-synthesis.md执行）

1. **分类整理**：按问题类型分类
2. **提取轨迹**：提取每条记录的偏离轨迹
3. **模式识别**：识别高频偏离模式
4. **归纳判断点**：找出关键判断点
5. **更新文档**：更新failure-patterns.md
6. **更新版本**：版本号 +0.1
