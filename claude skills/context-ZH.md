# Context-ZH: 中文语境增强

本技能确保 Claude 在对话中默认使用简体中文进行输出，并保持专业术语的准确性。

## 核心原则

1. **默认简体中文**：所有用户面向的文本输出均使用简体中文
2. **保留专业术语**：技术名词、函数名、变量名、API、SDK、库名等专业术语保持英文
3. **语义优化**：调用 translation skill 的语义优化能力

## 何时激活

- 用户使用中文（简体）进行任何交流时
- 用户明确提到"中文"、"简体中文"、"ZH"、"中文语境"
- 用户要求"用中文回答"、"请用中文解释"

## 语义优化流程

### 常见优化模式

**翻译腔避免**：
- ❌ "这个函数将会返回一个值" → ✅ "这个函数返回一个值"
- ❌ "被用于" → ✅ "用于"
- ❌ "是一个用来..." → ✅ "用于..."

**专业术语处理**：
- ✅ 保留英文术语：API、SDK、HTTP、Git、Python
- ✅ 使用公认中文译名：机器学习、区块链、持续集成
- ✅ 必要时括号注释：Webhook（网络钩子）

**简洁表达**：
- ❌ "请参阅文档" → ✅ "见文档"
- ❌ "如果有任何问题" → ✅ "如有疑问"

## 专业术语保留规则

| 类别 | 示例 | 处理 |
|------|------|------|
| 编程语言 | Python, JavaScript, Go | 保留英文 |
| 技术框架 | React, Vue, Django | 保留英文 |
| 协议标准 | HTTP, TCP/IP, WebSocket | 保留英文 |
| 开发工具 | Git, Docker, Kubernetes | 保留英文 |

## 质量检查

- [ ] 所有面向用户的文本都是简体中文
- [ ] 专业术语保持英文原样
- [ ] 无明显翻译腔
- [ ] 语义清晰，无歧义
- [ ] 标点符号使用正确
