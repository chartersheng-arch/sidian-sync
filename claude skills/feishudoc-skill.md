# Audio 需求文档格式规范

本 skill 用于生成符合ODM 内部规范的 Audio 类需求文档，输出格式兼容飞书文档。

## 文档标题

| 格式 | 示例 |
|------|------|
| `Audio：功能名称` | `Audio：声音助手`、`Audio：杜比功能及认证` |
| `Audio_ 功能名称` | `Audio_ 扬声器清理功能详解` |

> **注意**：禁止使用 `Audio-` 或其他分隔符

---

## 文档结构

| 章节 | 标题 | 说明 |
|:----:|------|------|
| 1 | 变更履历 | 版本变更记录表 |
| 2 | 需求内容 | 核心需求描述 |
| 3 | 需求前置条件\输入 | 前置条件 |
| 4 | 需求制作方法 | 制作方法 |
| 5 | 需求输出信息 | 输出信息 |
| 6 | 测试与验证 | 测试用例 |
| 7 | 多供物料是否关联 | 关联关系 |
| 8 | 相关问题 | Q&A |

---

## 章节格式

### 1. 变更履历

**表格（5列）**

| 变更版本 | 修订人 | 修订日期 | 审阅 | 修改情况及原因描述 |
|:--------:|:------:|:--------:|:----:|:------------------|
| V1.0 | 姓名 | 2026.3.2 | Audio小组 | 初版制成 |

> 修订日期格式：`YYYY.M.D`

---

### 2. 需求内容

#### 2.1 文档适用需求列表

**表格（2列）**

| 内部需求编号 | 需求名 |
|:-------------|:-------|
| 【S_IR_2741】 | 【FC】【声音助手】允许多声音 |
| 【S_IR_2741】【S_SR_2745】 | 【FC】【声音助手】允许多声音 |

> 需求编号使用【】包裹；需求名包含【FC】或【OEM】前缀

#### 2.2 需求原始描述

直接填写，包含：功能说明、步骤、有需求/无需求对比等

#### 2.3 需求新解

**2.3.1 原理解释**

- 软件时序图（标题格式：`xxx软件时序图`）
- OpenGrok 代码链接：`https://opengrok-lc-w.dun.mi.com/opengrok-w-lc/xref/...`

**2.3.2 需求内容**

- 子功能列表、UI 界面说明
- 关键说明使用「」包裹：`「通知音量控制」开关`

---

### 3. 需求前置条件\输入

**3.1 需求前置条件**

- 示例：`MI OS 业务适配`
- 示例：`MIUI业务集成，默认支持：设置->声音与震动->更多的声音设置`

**3.2 需求输入**

- 设计文档链接（飞书）：`https://mi-p.feishu.cn/docx/...`

---

### 4. 需求制作方法

**4.1 平台 QCOM**

**4.2 平台 MTK**

> 按平台分别填写制作方法

**属性配置示例：**
```
PRODUCT_PROPERTY_OVERRIDES += ro.vendor.audio.clean_speaker=true
```

**代码参考示例：**
```
C++参考提交：https://gerrit.odm.mioffice.cn/c/...
```

---

### 5. 需求输出信息

填写输出信息，如：`NA` 或 `UI` 相关描述

---

### 6. 测试与验证

#### 6.1 自测用例

**表格（3列）**

| 描述 | 步骤 | 预期结果 |
|:-----|:-----|:---------|
| 允许多声音 | 设置中打开开关，播放小米音乐，至后台后播放小米视频 | 两个应用是否同时发声 |
| 多应用音量调节 | 打开多个apk调节音量 | 多应用音量可单独调节 |

#### 6.2 LOG

```
Bash//命令说明
adb root
adb shell "getprop |grep xxx"

日志关键词：
status_speaker_clean|spkSV|kv_xml_parser

说明：
status_speaker_clean —— 扬声器清理参数下发
spkSV —— PA算法场景模式
```

#### 6.3 测试用例

- 6.3.1 FC测试用例 → `Audio_FC_测试用例`
- 6.3.2 Fulltest测试用例 → `Audio_Fulltest_测试用例`

---

### 7. 多供物料是否关联

`是` 或 `否`

---

### 8. 相关问题

**BUG 记录模板：**
```
BUG编号 [平台][模块][类型][优先级] 简要描述

[问题现象]：xxx
[问题原因]：xxx
[解决方法]：xxx
[修复提交]：https://gerrit.xxx.cn/c/...
```

**澄清模板：**
```
平台澄清：（不能修复）
一、问题分析：xxx
二、问题总结：xxx
三、备注：xxx
```

## 特殊格式
